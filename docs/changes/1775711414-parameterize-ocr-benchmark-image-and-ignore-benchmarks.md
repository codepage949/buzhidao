# OCR 벤치마크 이미지 경로화 및 산출물 무시 규칙 정리

OCR 테스트가 저장소에 고정된 `test.jpg` 파일에 의존하지 않도록 외부 이미지 경로를 환경 변수로 받게 변경하고, 비교 및 벤치마크 과정에서 생성되는 산출물이 Git 추적 대상에 섞이지 않도록 정리합니다.

## 구현 상세

### 1. OCR 통합 테스트의 외부 이미지 경로화
- `src/ocr/mod.rs`의 벤치마크 성격 테스트가 `BUZHIDAO_OCR_BENCH_IMAGE` 환경 변수를 통해 입력 이미지를 받도록 변경.
- 환경 변수가 없거나 파일이 존재하지 않으면 테스트를 건너뛰도록 처리해 로컬 환경 차이로 인한 불필요한 실패를 방지.
- 기존 `test.jpg` 고정 의존성을 제거하고, 테스트 기대값도 특정 샘플 이미지 전용 하한 개수 비교 대신 "최소 1개 이상 검출"로 일반화.

### 2. 비교 스크립트 사용법 정리
- `scripts/compare_onnx.py`의 사용 예시를 특정 파일명 대신 일반 이미지 경로 인자로 수정해 외부 벤치마크 이미지 기반 워크플로와 일치시킴.

### 3. 생성 산출물 추적 방지
- `.gitignore`에 `benchmarks/`를 추가해 비교 실험이나 벤치마크 중 생성되는 파일이 작업 트리에 섞이지 않도록 정리.
- 저장소 루트의 `test.jpg`는 더 이상 기본 테스트 자산이 아니므로 삭제.

## 테스트 및 검증
- `cargo test`로 OCR 테스트 스위트를 실행해 환경 변수 미설정 시 벤치마크 테스트가 정상적으로 skip 처리되는지 확인.
