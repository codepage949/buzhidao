# OCR 결과 확인 후 영역 재선택 회귀 수정

## 배경

오버레이에서 영역을 선택해 OCR 결과를 본 뒤,
다시 빈 영역을 드래그해서 새 영역을 선택하려고 하면
선택 모드로 들어가기 전에 오버레이가 닫히는 회귀가 있었다.

원인은 OCR 결과 상태(`ready`/`error`)에서 빈 영역 `click`이
곧바로 `close_overlay`로 이어지고,
드래그 재선택 시작을 위한 상태 전환이 그보다 늦게 처리되기 때문이다.

추가로, CPU 모드에서는 첫 번째 영역 OCR 시 보관 중인 원본 캡처를
`take()`로 소모하고 있어 두 번째 영역 선택부터는
`선택할 캡처 이미지가 없음` 오류가 발생했다.

## 목표

1. OCR 결과 확인 후에도 빈 영역 드래그로 다시 선택 모드에 진입할 수 있게 한다.
2. 드래그 시작에 수반되는 클릭 꼬리 이벤트가 오버레이 닫기로 이어지지 않게 한다.
3. 선택 크기가 너무 작아 무효가 된 경우에는 직전 결과 상태를 복구한다.
4. 같은 오버레이 세션에서 여러 번 영역 OCR을 수행할 수 있게 원본 캡처를 유지한다.
5. 빈 영역을 이동 없이 한 번 클릭해 한 점으로 끝난 경우에는 오버레이를 닫는다.

## 변경 내용

### `ui/src/overlay.tsx`

- `ready`/`error` 상태에서도 마우스 다운으로 재선택을 시작할 수 있게 변경
- 재선택 진입 전 상태를 `ref`로 보관했다가 무효 선택 시 복구
- 선택 시작 시 다음 빈 클릭 1회를 무시하도록 닫기 억제 플래그를 갱신
- 선택 완료 시 결과가 `점`이면 오버레이를 닫고, 작은 사각형이면 직전 결과 상태로 복귀

### `ui/src/overlay_close.ts`

- `selection_started` 이벤트를 추가
- 영역 선택 시작 시 후속 클릭이 닫기로 처리되지 않도록 억제 규칙 확장

### `ui/src/overlay_close_test.ts`

- OCR 결과 상태에서 재선택 시작 시 닫기 억제가 켜지는지 테스트 추가

### `ui/src/overlay_selection.ts`

- 최종 선택 결과를 `close/resume/submit`으로 판정하는 순수 함수 추가
- 점 클릭과 작은 사각형, 유효 사각형을 분리해 처리

### `ui/src/overlay_selection_test.ts`

- 점 클릭은 닫기, 작은 사각형은 복귀, 유효 사각형은 OCR 실행으로 판정되는지 테스트 추가

### `src/lib.rs`

- 영역 OCR 시 pending capture를 `take()`하지 않고 clone해서 사용
- 오버레이를 닫기 전까지 같은 원본 캡처로 여러 번 영역 선택 가능
- pending capture 재사용 헬퍼와 단위 테스트 추가

### `src/services.rs`

- `CaptureInfo`에 `Clone`을 추가해 동일 캡처 재사용 가능하게 변경
