# 이전 OCR 종료 대기 중 재캡처 요청 예약

## 배경

오버레이를 닫아도 이미 시작한 OCR은 바로 중단되지 않는다.
현재 구현은 이 구간에서 사용자가 캡처 단축키를 다시 눌러도 입력을 그냥 무시한다.

이 때문에 사용자는 "닫았는데 왜 다시 안 뜨지?"라는 상태가 되고,
이전 OCR이 끝난 뒤 다시 단축키를 눌러야 한다.

## 결정

이전 OCR 작업이 아직 끝나지 않은 동안 단축키가 다시 눌리면,
요청을 한 건만 예약한다.

이때 오버레이를 다시 보여주고
`"이전 작업이 끝나면 새 캡처를 시작합니다"` 안내를 표시한다.
이전 OCR이 끝나면 예약된 요청을 자동으로 실행해 기존 캡처 흐름으로 진입한다.

초기 warmup처럼 OCR이 실제로 돌고 있지 않은 `busy` 상태에서는
잘못된 안내가 뜨지 않도록 예약을 만들지 않는다.

## 변경 사항

### 런타임
- `app/src/lib.rs`
  - 실제 OCR 진행 여부와 예약 재요청 여부를 관리하는 상태를 추가.
  - 단축키 재입력 시 `overlay visible / idle / OCR 진행 중 / warmup busy`를 구분하는 분기를 추가.
  - OCR 종료 후 예약된 재요청이 있으면 자동으로 `handle_prtsc(...)`를 다시 태운다.
  - 오버레이를 닫을 때 예약된 재요청은 함께 취소한다.
- `app/src/platform.rs`
  - 전역 단축키 레벨에서는 `overlay가 이미 보이는 경우`만 차단하고,
    `busy` 중 재요청 판단은 `lib.rs`로 넘긴다.
  - 안내 문구만 보여주는 `show_overlay_notice(...)` 헬퍼를 추가.

### UI
- `app/ui/src/overlay.tsx`
  - `"이전 작업이 끝나면 새 캡처를 시작합니다"` 같은 안내 문구를 표시하는 `pending_retry` 상태를 추가.
  - 예약 재요청 이벤트를 받으면 오버레이를 다시 표시하고,
    기존 OCR 완료 이벤트가 오면 자동으로 일반 로딩/결과 상태로 전환한다.

## 테스트

- `cd app && cargo test`
- `cd app/ui && deno test`

## 수동 검증

- OCR 로딩 중 오버레이를 닫은 뒤 단축키를 다시 누르면 안내 오버레이가 뜨는지.
- 이전 OCR 종료 후 새 캡처가 자동으로 시작되는지.
- 안내 오버레이를 다시 닫으면 예약된 재요청도 취소되어 자동 재캡처가 발생하지 않는지.
- 앱 시작 직후 warmup 중 단축키를 눌러도 잘못된 종료 대기 안내가 뜨지 않는지.
