# 팝업 닫기 버튼 동작 수정

## 변경 목적

`ui/src/popup.tsx`의 닫기 버튼이 안정적으로 동작하지 않았다.

팝업 닫기 버튼과 ESC는 팝업만 닫아야 하고 오버레이는 유지되어야 하므로,
팝업 전용 Rust 커맨드로 닫기 경로를 분리해 동작을 안정화한다.

## 구현 계획

1. 팝업 닫기 버튼과 ESC 키가 팝업 전용 Rust 커맨드를 호출하도록 정리한다.
2. 중복 로직이 있으면 최소한으로 정리한다.
3. 프런트엔드와 Rust 빌드 검증으로 회귀가 없는지 확인한다.

## 구현 사항

- `ui/src/popup.tsx`
  - 닫기 버튼 동작을 `invoke("close_popup")`로 변경한다.
  - ESC 키 처리와 버튼 클릭이 동일한 `close` 함수 경로를 사용하도록 정리한다.
- `src/lib.rs`
  - `close_popup` 커맨드를 추가해 팝업만 숨기고 오버레이 포커스를 복구하도록 했다.

## 테스트 계획

- `cargo tauri dev` 실행 후 팝업 닫기 버튼 클릭 시 팝업만 닫히고 오버레이는 유지되는지 수동 확인한다.
- 프런트엔드와 Rust 빌드 검증이 통과하는지 확인한다.

## 테스트 결과

- `deno task --config ui/deno.json build`
  - 프런트엔드 빌드가 정상적으로 완료되는 것을 확인했다.
- `cargo check`
  - Rust 커맨드 추가 후에도 컴파일이 정상적으로 완료되는 것을 확인한다.
- 팝업 닫기 버튼 수동 확인
  - Tauri 런타임에서 실제 팝업을 띄운 뒤 확인해야 하는 항목이라 자동화하지 않았다.

## 리팩토링 검토

- 닫기 버튼과 ESC 키 처리 경로를 `close` 함수 하나로 맞춰 중복을 줄였다.
- 추가 리팩토링은 필요하지 않았다.
