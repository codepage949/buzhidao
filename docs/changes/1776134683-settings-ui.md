# 설정 UI (UserSettings 윈도우)

## 목적

사용자 관점 항목을 런타임 UI에서 편집·저장할 수 있게 한다.
개발 모드와 배포 모드 모두 `.env`를 참조하고 저장한다.
차이는 경로와 초기 생성 방식이다.
인프라성 값(`OCR_SERVER_*`, 타임아웃, debug 플래그)은 제외.

## 노출 항목

| 키                  | UI 컨트롤               | 검증              |
| ------------------- | ----------------------- | ----------------- |
| `SOURCE`            | 라디오 (en / ch)        | enum              |
| `SCORE_THRESH`      | 슬라이더 0.0–1.0        | clamp [0,1]       |
| `OCR_SERVER_DEVICE` | 라디오 (cpu / gpu)      | enum, GPU 실행 파일일 때만 노출 |
| `AI_GATEWAY_API_KEY`| password input          | trim, 비밀        |
| `AI_GATEWAY_MODEL`  | text input              | trim, non-empty   |
| `WORD_GAP`          | number input            | i32 ≥ 0           |
| `LINE_GAP`          | number input            | i32 ≥ 0           |
| system prompt       | textarea                | trim              |

## 설계

### 저장 위치

개발 모드: 프로젝트의 `app/.env`

배포 모드: `app_data_dir/.env`

- 개발 모드에서는 프로젝트의 `.env`를 읽고 그대로 갱신한다.
- 배포 모드에서는 `app_data_dir/.env`를 읽고 갱신한다.
- 배포 모드에서 `.env`가 없으면 빌드 시 포함된 `.env.example` 스냅샷으로 생성한다.
- 배포 모드에서 `.env`가 이미 있으면 `.env.example`을 기준으로 누락 키를 채우고, 기존 `.env` 값이 예시값을 override한 최종 `.env`를 다시 생성한다.

### 상태 관리

- `Config`를 `Arc<RwLock<Config>>`로 변경.
- 기존 `state::<Config>()` 호출부는 `.read().clone()`으로 스냅샷.
- 저장 시 `.write()`로 필드 갱신 후 JSON persist.

### 반영 시점

- AI/OCR 파이프라인 값(`source`, `score_thresh`, `ai_gateway_*`, prompt, gap): 즉시 반영.
- `ocr_server_device`: GPU 실행 파일(`ocr_server_gpu`)일 때만 노출. 저장은 허용하되 사이드카 재시작 필요 — 다음 앱 실행 시 적용.

### 진입점

트레이 메뉴 "설정…" 선택 시 `settings` 윈도우 show + focus.

## 파일

- `app/src/settings.rs` (신규): `UserSettings`, `.env` 저장/병합, `apply_to`, `validate`.
- `app/src/config.rs`: `Config`는 그대로 두고, settings 모듈이 user-editable 필드를 덮어쓴다.
- `app/src/lib.rs`: 상태를 `Arc<RwLock<Config>>`로 교체. `get_user_settings`, `save_user_settings` 커맨드, settings 파일 경로 상태, 트레이 메뉴 진입점 추가.
- `app/tauri.conf.json`, `capabilities/default.json`: `settings` 윈도우 추가.
- `app/ui/src/settings.html` + `settings.tsx` (신규): 폼 UI.
- `app/ui/vite.config.ts`: 엔트리 추가.

## 구현

- 개발 모드에서는 프로젝트 `.env`를 그대로 로드하고 저장한다.
- 배포 모드에서는 `app_data_dir/.env`를 materialize한 뒤 그 파일을 로드하고 저장한다.
- 배포 모드 `.env` materialize는 `.env.example`을 기준으로 하고, 기존 `.env` 값이 있으면 그 값으로 override한다.
- 저장 시 공유 `Config`에는 즉시 반영하고, 파일 persist는 `.env`에 수행한다.
- `ocr_server_device` 변경은 저장은 허용하되, 저장 응답에 `restart_required`를 포함해 UI에서 재시작 필요 문구를 표시한다.
- CPU 실행 파일 경로에서는 장치 설정 UI를 숨기고, GPU 실행 파일 경로에서만 장치 선택을 보여준다.
- 설정 창은 트레이 메뉴 "설정…"으로 열고, ESC/닫기 버튼으로 숨긴다.
- UI는 source/device 라디오, score 슬라이더, API key/model, gap 숫자 입력, system prompt textarea를 제공한다.

## 테스트

- `settings::validate`: score_thresh clamp, source/device enum, gap 음수, trim 보정.
- `settings::apply_to`: 필드가 Config에 전부 반영된다.
- `settings::materialize_env_file`: 배포 모드 `.env`를 `.env.example` 기준으로 생성/보정한다.
- `settings::save_to_env_file`: `.env`의 관리 키를 갱신한다.
- serde round trip.
- 기존 UI 단위 테스트는 회귀 확인용으로 유지.
- 설정 창 자체는 육안 검증 대상.
