# 앱 단일 인스턴스 실행 보장

## 구현 목적

동일한 프로그램이 동시에 여러 개 실행되지 않도록 제한해 트레이 아이콘 중복, 전역 키 훅 중복 등록, 백그라운드 작업 경쟁을 방지한다.

- 두 번째 실행 시 새 프로세스가 그대로 올라오지 않도록 막는다.
- 이미 실행 중인 인스턴스가 화면에 표시 중인 창이 있으면 해당 창으로 포커스를 복구한다.
- 기존 OCR/오버레이 흐름은 유지하고 진입점에서만 제어한다.

## 구현 계획

1. 현재 Tauri 진입점에 단일 인스턴스 플러그인을 추가할 위치를 정한다.
2. 기존 인스턴스에서 어떤 창을 앞으로 가져올지 결정하는 로직을 분리한다.
3. 순수 로직 단위 테스트를 추가해 포커스 대상 선택 규칙을 검증한다.
4. `cargo test`, `cargo build`로 회귀를 확인한다.

## 구현 사항

- `app/Cargo.toml`에 `tauri-plugin-single-instance` 의존성을 추가했다.
- `app/src/lib.rs`에서 Tauri 빌더에 단일 인스턴스 플러그인을 등록했다.
- 두 번째 실행이 들어오면 기존 인스턴스에서 현재 표시 중인 창을 찾아 앞으로 가져오도록 연결했다.
- `app/src/window.rs`에 포커스 대상 선택 헬퍼와 테스트를 추가했다.

## 테스트 결과

- `cargo test`
  - 테스트 9개 통과
- `cargo build`
  - 개발 빌드 성공

## 추가 검토

- 포커스 대상은 현재 표시 중인 UI가 있는 경우에만 복구한다.
- 숨김 상태의 트레이 앱을 강제로 노출하지는 않는다. 필요 시 추후 두 번째 실행을 “메인 동작 실행”으로 확장할 수 있다.
