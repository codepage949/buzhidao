# CPU 모드 이미지 전처리 병렬화

## 배경

현재 CPU OCR 경로는 `cls`/`rec` 전처리 자체는 rayon으로 병렬화돼 있지만,
그 이전 단계인 crop 생성과 회전 적용은 여전히 직렬로 처리하고 있다.

이 구간은 박스 수가 많을수록 비용이 누적되고,
세션 공유나 추론 상태와 무관한 순수 이미지 처리이므로
안전하게 병렬화할 수 있다.

## 목표

1. CPU 모드에서 박스 crop 생성 비용을 줄인다.
2. `cls` 결과에 따른 `rotate180` 적용 비용도 병렬화한다.
3. 세션 동시 추론처럼 위험도가 높은 병렬화는 건드리지 않는다.

## 변경 내용

### `src/ocr/mod.rs`

- `merged_boxes -> crop_box` 생성 과정을 helper로 분리하고 rayon 병렬화
- `cls` 결과를 반영한 `rotate180` 적용도 helper로 분리하고 rayon 병렬화
- 작은 입력에서는 병렬화 오버헤드를 피하도록 1개 입력은 직렬 처리
- helper 테스트를 추가해 순서 보존과 회전 적용을 확인

## 기대 효과

- 박스 수가 많은 CPU 영역 OCR에서 `cls`/`rec` 앞단 이미지 처리 시간을 줄인다.
- 세션 락 구조를 유지하므로 추론 안정성에는 영향을 주지 않는다.
