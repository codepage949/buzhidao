# GitHub Pages 소개 페이지 제작

스크롤 애니메이션을 통해 앱의 핵심 기능(OCR 인식 및 실시간 번역)을 시각화하는 현대적이고 세련된 GitHub Pages 랜딩 페이지를 제작합니다.

## 구현 상세

### 1. 여유로운 스크롤 기반 시뮬레이션
- **Sticky 기반 애니메이션**: `900vh`에 달하는 깊은 스크롤을 활용하여 사용자가 애니메이션의 호흡을 직접 제어할 수 있도록 구현.
- **단계별 타임라인**:
    - **Entry**: 히어로 섹션 페이드아웃 및 macOS 스타일 윈도우 프레임 상승 진입.
    - **Pause**: 윈도우 안착 후 사용자가 화면을 인지할 수 있는 충분한 대기 시간 부여.
    - **Scan & Loading**: 광채 효과가 있는 스캔 라인 이동과 실제 앱의 "텍스트 인식 중" 로딩 시뮬레이션.
    - **Recognition**: 타겟 단어("不知道")에 정교하게 정렬된 OCR 박스 생성 및 맥동(Pulse) 효과.
    - **Interaction**: 가상 마우스 커서가 부드럽게 이동하여 단어를 클릭하는 액션.
    - **Result**: 글래스모피즘 번역 팝업 등장 및 이모지 깨짐 방지 로직이 적용된 "몰루🤪" 타이핑 효과.

### 2. 현대적 디자인 (Modern & Playful)
- **Glassmorphism**: 윈도우 프레임 및 팝업에 블러와 투명도 효과를 적용하여 세련된 UI 제공.
- **Mesh Gradient**: 배경에 다채로운 빛이 부드럽게 움직이는 효과를 주어 생동감 부여.
- **Typography**: 'Plus Jakarta Sans'와 'Noto Sans KR'을 조합한 모던한 타이포그래피.
- **GitHub 연동**: 우측 상단에 고정된 GitHub 아이콘 및 푸터 링크를 통해 프로젝트 접근성 강화 (`https://github.com/codepage949/buzhidao`).

## 테스트 및 검증
- **브라우저 호환성**: Chrome, Edge 등 주요 브라우저에서 스크롤 애니메이션의 매끄러움 확인.
- **반응형 대응**: 모바일 뷰포트에서의 레이아웃 조정 및 텍스트 가독성 검증.
- **유니코드 처리**: 이모지가 포함된 문자열이 타이핑 중에 깨지지 않고 정상 출력되는지 확인.
