# CPU 모드 cls·rec 소형 crop 처리 최적화

## 배경

현재 CPU 모드는 전체 화면 OCR 대신 사용자가 영역을 고른 뒤
잘린 이미지에 대해서만 OCR을 수행한다.

이 경로에서는 큰 det 비용보다 `cls + rec`의 작은 crop 다건 처리가
상대적으로 더 큰 병목이 된다.

특히 `cls`는 모든 crop에 대해 항상 전수 실행하고 있어,
일반적인 UI/문서 화면처럼 180도 뒤집힌 텍스트가 거의 없는 경우에도
불필요한 추론 비용이 발생한다.

## 목표

1. CPU 모드에서 작은 crop가 많을 때 `cls` 전수 비용을 줄인다.
2. 회전 텍스트가 실제로 섞인 경우에는 기존 정확도를 최대한 유지한다.
3. 기존 `rec` 배치 경로는 유지하되, `cls`의 낭비를 먼저 줄여 체감 속도를 개선한다.

## 변경 내용

### `src/ocr/mod.rs`

- CPU 모드에서 `cls` 전수 실행 전에 대표 crop만 샘플링해 검사한다.
- 샘플 검사 결과가 모두 정방향이면:
  - 전체 `cls`를 생략하고 모든 crop을 정방향으로 간주한다.
- 샘플 중 하나라도 180도 회전이 나오면:
  - 기존처럼 전체 `cls` 배치 실행으로 폴백한다.
- 샘플 인덱스 선택과 전수 실행 여부를 별도 헬퍼로 분리한다.
- 로그에 `cls 샘플` 또는 `cls 전수` 여부를 남겨 실제 동작을 확인할 수 있게 한다.

## 기대 효과

- CPU 모드에서 작은 crop가 많이 생기는 일반 화면 OCR의 `cls` 시간을 줄인다.
- 회전 텍스트가 드문 일반 UI 캡처에서는 `cls` 비용을 거의 없앨 수 있다.
- `rec` 경로는 그대로 두므로 회귀 범위를 제한할 수 있다.
