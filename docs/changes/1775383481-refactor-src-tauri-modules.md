# src Tauri 모듈 리팩토링

## 리팩토링 목적

`src/lib.rs`가 설정 로드, OCR, AI 호출, 팝업 좌표 계산, 창 제어, Tauri 부팅까지 모두 포함하고 있어 변경 영향 범위를 파악하기 어렵다.

- 중복 제거: 팝업 위치 계산 로직을 단일 순수 함수로 통합한다.
- 모듈화: 설정, 서비스, 창 제어, 팝업 좌표 계산을 파일 단위로 분리한다.
- 유지보수성 개선: `lib.rs`는 Tauri 엔트리와 흐름 제어에 집중시킨다.

## 리팩토링 계획

1. `Config`와 환경변수 로드를 별도 모듈로 분리한다.
2. OCR/AI 네트워크 로직과 캡처 타입을 서비스 모듈로 이동한다.
3. 팝업 좌표 계산을 재사용 가능한 순수 함수 하나로 통합한다.
4. `lib.rs`는 커맨드와 앱 부팅 흐름만 남기도록 정리한다.
5. `cargo test`, `cargo check`로 회귀를 확인한다.

## 리팩토링 사항

- `src/config.rs`를 추가해 환경변수 로드와 `Config` 구성을 분리했다.
- `src/services.rs`를 추가해 화면 캡처, OCR 요청, AI 번역 요청을 한곳에 모았다.
- `src/popup.rs`를 추가해 팝업 좌표 계산을 단일 순수 함수로 통합하고 관련 테스트를 이동했다.
- `src/window.rs`를 추가해 창 숨김과 포커스 복구 유틸을 분리했다.
- `src/lib.rs`는 Tauri 커맨드, PrintScreen 처리 흐름, 앱 부팅 코드만 남도록 축소했다.

## 테스트 결과

  - 테스트 4개 통과
- `cargo check`
  - 컴파일 검증 통과

## 추가 검토

- 현재 `src` 기준으로는 설정, 서비스, 좌표 계산, 창 제어 책임 분리가 완료돼 추가 분해 이득이 크지 않다.
- 다음 리팩토링 후보는 네트워크 클라이언트 재사용이나 에러 타입 정교화지만, 이번 범위의 핵심인 중복 제거와 모듈화는 충족됐다.
