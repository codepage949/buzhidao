# CPU OCR 영역 선택 지원

## 배경

CPU 환경에서 전체 화면 OCR은 대기 시간이 길다.
GPU가 없는 경우에는 먼저 사용자가 관심 영역만 선택하고,
선택된 부분에만 OCR을 태우는 편이 체감 성능에 유리하다.

## 구현 계획

1. CPU OCR 경로를 런타임에서 구분한다.
2. CPU 경로에서는 전체 캡처 이미지를 보관하고 오버레이에 영역 선택 상태를 띄운다.
3. 선택된 영역만 crop 해서 OCR을 실행하고 결과 좌표를 전체 화면 기준으로 다시 보정한다.
4. 좌표 변환과 crop 로직을 테스트한다.

## 구현 사항

- `OcrEngine`이 현재 실제 GPU 세션을 사용하는지 추적하고 `is_cpu_mode()`를 노출한다.
- `handle_prtsc()`는 CPU 모드일 때 전체 OCR을 바로 실행하지 않고, 캡처 이미지를 앱 상태에 저장한 뒤 오버레이에 영역 선택 이벤트를 보낸다.
- 새 Tauri 커맨드 `run_region_ocr`를 추가해 오버레이에서 전달한 CSS 좌표를 실제 캡처 이미지 좌표로 변환한 뒤 해당 영역만 OCR한다.
- OCR 결과는 crop 오프셋만큼 다시 더해서 기존 오버레이 렌더링 구조를 그대로 재사용한다.
- 오버레이 UI에 드래그 선택 상태를 추가하고, CPU 모드에서만 영역 선택 박스와 안내 문구를 표시한다.
- 오버레이를 닫을 때 보관 중이던 캡처 이미지를 정리한다.
- 드래그 종료 직후 같은 클릭 이벤트로 오버레이가 닫히지 않도록 다음 close 클릭을 한 번 무시한다.
- `rec` 배치 인식은 너비 편차가 과한 crop들을 같은 청크로 묶지 않도록 분할 기준을 추가한다.
- CPU 모드의 `rec`는 폭 상한을 더 낮게(`960`) 적용하고 low-score single retry를 생략해 긴 줄 인식 비용을 줄인다.
- CPU 모드의 `det`는 선택 영역이 작을 때 `resize_long=1024`까지 불필요하게 업스케일하지 않는다.
- CPU 모드의 `rec`는 작은/중간/긴 crop에 대해 배치 크기를 더 보수적으로 잡아 CPU 배치 추론 폭주를 줄인다.
- CPU 모드의 작은 박스 필터는 인식률 하락 대비 이득이 작아 적용하지 않는다.
- CPU 모드에서는 같은 줄의 인접 박스를 보수적으로 병합해 `rec` 입력 수 자체를 줄인다.
