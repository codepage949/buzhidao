# Linux 로딩 창 높이 고정

## 배경

Linux에서 초기 로딩 창 높이가 Windows보다 2~3배 크게 보이는 문제가 보고되었다.
Linux WebKitGTK 환경에서는 창 자체의 최소 inner height가 기대보다 크게 잡힐 수 있어,
창 크기보다 실제 보이는 로딩 박스의 크기를 안정화하는 방향으로 처리한다.

## 계획

1. Linux에서 실제 창 높이를 줄일 수 없는 경우에도 보이는 로딩 박스가 커지지 않게 한다.
2. 로딩 창은 투명하게 만들고, 실제 배경은 300x90 중앙 패널에만 적용한다.
3. 앱 시작 직후 로딩 창을 직접 표시하도록 초기 `visible` 설정을 끈다.
4. Tauri 설정과 HTML 렌더링 계약이 어긋나지 않도록 단위 테스트를 추가한다.

## 구현

- `tauri.conf.json`의 로딩 창 초기 표시를 끄고, Rust setup에서 직접 표시하도록 바꿨다.
- 로딩 창을 투명 창으로 바꿔 큰 WebView 배경이 보이지 않게 했다.
- `ui/src/loading.html`은 viewport 전체 높이 기준으로 내용을 중앙 정렬하도록 유지한다.
- Linux WebKitGTK가 WebView 최소 높이를 강제하는 경우를 대비해 창 배경은 투명하게 두고, 실제 보이는 로딩 배경을 300x90 중앙 패널로 렌더링한다.
