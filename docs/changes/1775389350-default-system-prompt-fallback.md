# SYSTEM_PROMPT_PATH 기본값 제거 및 기본 프롬프트 폴백

## 구현 목적

`SYSTEM_PROMPT_PATH` 환경변수에 암묵적인 기본 파일 경로를 두지 않고, 미설정 시 내장 기본 프롬프트를 사용하도록 동작을 단순화한다.

- 환경변수가 없을 때 `.system_prompt.txt` 파일 존재 여부에 앱 동작이 좌우되지 않게 한다.
- 기본 프롬프트를 코드에 명시해 초기 실행 구성을 단순하게 만든다.
- 설정 문서와 예시 환경변수를 실제 동작과 일치시킨다.

## 구현 계획

1. 시스템 프롬프트 로딩 로직에서 `SYSTEM_PROMPT_PATH` 미설정 시 기본 프롬프트 문자열로 폴백하도록 수정한다.
2. `.env.example`와 `README.md`에서 `SYSTEM_PROMPT_PATH`를 선택 환경변수로 문서화한다.
3. 설정 로직 테스트를 추가해 기본 프롬프트 폴백과 파일 로딩 동작을 검증한다.

## 구현 사항

- `app/src/config.rs`에서 기본 시스템 프롬프트를 `"다음을 한국어로 번역하세요."` 상수로 정의했다.
- `SYSTEM_PROMPT_PATH`가 없으면 내장 기본 프롬프트를 반환하고, 있으면 지정 파일을 읽도록 로직을 변경했다.
- 환경변수 기반 분기 로직에 대한 단위 테스트 2개를 추가했다.
- `app/.env.example`, `app/README.md`에서 `SYSTEM_PROMPT_PATH`를 선택 항목으로 바꾸고 기본 프롬프트 폴백 동작을 명시했다.

## 테스트 결과

- `cargo test config`
  - `app/` 기준 실행, 테스트 2개 통과

## 추가 검토

- 현재 범위에서는 시스템 프롬프트 문자열이 짧고 고정되어 있어 별도 리팩토링 이득이 크지 않다.
- 향후 프롬프트 선택지가 늘어나면 설정 소스를 enum 등으로 분리하는 리팩토링을 검토할 수 있다.
