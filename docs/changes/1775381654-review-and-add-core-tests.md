# Rust 핵심 로직 테스트 보강

## 변경 목적

현재 프로젝트 테스트 현황을 점검한 결과:

- `ui/src/detection_test.ts`에서 프런트엔드 핵심 그룹화 로직을 검증 중이다.
- `ocr/test_main.py`에서 OCR 서버 핵심 경로를 검증 중이다.
- Rust 앱(`src/lib.rs`)은 핵심 순수 로직 테스트가 비어 있다.

팝업 위치 계산은 UI 동작에 직접 영향을 주는 핵심 로직이면서도
순수 함수로 분리해 안정적으로 테스트할 수 있으므로 이를 우선 보강한다.

## 구현 계획

1. 팝업 위치 계산 로직을 테스트 가능한 순수 함수로 분리한다.
2. Rust 단위 테스트를 추가해 대표 배치 경계 조건을 검증한다.
3. 기존 UI/OCR 테스트는 유지하고 전체 테스트 명령을 통과시키는지 확인한다.

## 구현 사항

- `src/lib.rs`
  - 팝업 위치 계산 규칙을 테스트할 수 있도록 `calc_popup_pos_from_screen` 헬퍼를 추가했다.
  - 다음 경계 조건을 검증하는 Rust 단위 테스트 4개를 추가했다.
    - 오른쪽 공간이 충분한 경우
    - 오른쪽 공간이 부족한 경우
    - 왼쪽도 부족해 `x = 0`으로 고정되는 경우
    - 화면 아래를 벗어나 `y`가 보정되는 경우
- 테스트 현황 점검 결과
  - UI: `ui/src/detection_test.ts`가 핵심 텍스트 그룹화 로직을 이미 충분히 검증하고 있어 추가 보강이 필요하지 않았다.
  - OCR: `ocr/test_main.py`가 업로드 저장과 `/infer/{src}` 주요 경로를 이미 검증하고 있어 코드 추가보다 실행 가능 여부 점검이 우선이었다.

## 테스트 결과

- `cargo test`
  - Rust 단위 테스트 4개가 모두 통과했다.
- `deno test --config ui/deno.json ui/src/detection_test.ts`
  - 기존 UI 테스트 11개가 모두 통과했다.
- `uv run pytest`
  - 현재 로컬 환경이 CPython 3.14.2라 `paddlepaddle-gpu==3.2.2`의 `cp313` 전용 wheel과 맞지 않아 실행하지 못했다.
  - OCR 테스트 코드 자체는 이미 존재하며, 실행 환경은 Python 3.13 계열로 맞춰야 한다.

## 리팩토링 검토

- 테스트를 위해 순수 계산 로직을 별도 함수로 추가해 경계 조건 검증이 가능하도록 정리했다.
- UI/OCR 쪽은 추가 리팩토링 없이 기존 테스트 구조를 유지하는 편이 적절했다.
