# OCR 중복 제거 근본 개선 — det 단계 타일 우선 NMS로 사후 휴리스틱 제거

## 배경

타일 OCR에서 같은 텍스트 영역에 det가 여러 번 독립 실행되어 중복이 생기고,
이미 rec까지 돌린 뒤 텍스트+기하 휴리스틱으로 제거하는 구조였다.
rec 결과가 조금만 달라도 휴리스틱이 빗나가는 근본적 한계가 있었다.

## 수정

### 파이프라인 구조 변경

기존:
```
전체 이미지 → [det→cls→rec] → detections
각 타일    → [det→cls→rec] → tile detections
전체 병합  → 사후 텍스트+기하 휴리스틱 중복 제거
```

변경 후:
```
전체 이미지 → det → full boxes
각 타일    → det → tile boxes (좌표 복원)
타일끼리   → NMS (인접 타일 오버랩 중복 제거)
전체 박스  → 타일이 커버하는 영역은 제거, 못 잡은 영역만 보충
고유 박스  → cls+rec (원본 이미지에서 1회만)
```

### 1. `OcrEngine` — det/rec 단계 분리

`predict()`를 `detect()` + `recognize_boxes()`로 분리했다.
- `detect()`: det만 실행하여 `DetBox` 반환
- `recognize_boxes()`: 주어진 박스에 cls+rec 실행
- `predict()`: 편의 메서드로 테스트 전용(`#[cfg(test)]`)

### 2. `predict_with_tiles` — 타일 우선 det 병합 → rec

타일 det이 더 높은 유효 해상도로 실행되��로 타일 박스를 우선한다.
전체 이미지 det 박스는 타일이 커버하지 않는 영역에서만 보충용으로 사용한다.

### 3. `merge_tile_priority` — 타일 우선 병합

1. 타일 박스끼리 NMS (인접 타일 오버랩 중복 제거)
2. 전체 이미지 박스 중 타일 박스 면적의 50% 이상 겹치는 것은 제거
3. 타일이 못 잡은 영역만 전체 이미지 박스로 보충

타일을 우선하는 이유: 타일 det는 더 높은 유효 해상도로 실행되어
박스 위치가 더 정확하고, 작은 텍스트도 더 잘 잡는다.

### 4. `nms_boxes` — 순수 기하학적 NMS

면적 내림차순으로 정렬 후, 큰 박스 → 작은 박스 순서로 억제 판정:
- IoU > 0.5: 중복 (표준 NMS 임계값)
- 작은 박스 포함 비율 > 0.7: 큰 박스에 흡수

### 5. 사후 중복 제거 코드 제거

`deduplicate_detections`, `is_same_detection`, `should_replace` 및
그에만 쓰이던 유틸리티 함수를 모두 제거했다.

## 효과

- 중복 근본 차단: 같은 영역에 rec이 중복 실행되지 않음
- 인식률 보존: 타일 박스를 우선하여 높은 해상도 det 결과 유지
- 성능 개선: 중복 영역에 대한 cls+rec 연산 절약
- 코드 단순화: 텍스트+기하 복합 휴리스틱 → 기하학적 NMS + 타일 우선 병합

## 테스트

- `NMS_억제_판정_IoU가_높으면_true`
- `NMS_억제_판정_포함_비율이_높으면_true`
- `NMS_억제_판정_겹치지_않으면_false`
- `NMS는_IoU가_높은_박스를_억제한다`
- `NMS는_포함된_작은_박스를_억제한다`
- `NMS는_떨어진_박스를_유지한다`
- `NMS는_같은_줄_다른_단어를_유지한다`
- `타일_우선_병합은_타일이_커버하는_전체_박스를_제거한다`
- `타일_우선_병합은_타일이_못_잡은_영역을_전체_박스로_보충한다`

40개 전부 통과. 중국어 이미지 추론 비교 테스트 영향 없음.

### 6. services.rs — 타일 그리드 밀도 증가 (3x3/4x4)

기존 2x2/3x3 타일은 각 타일이 여전히 ~1000px 이상이라
resize_long=960 이후 실질적 해상도 이득이 미미했다 (~10.6px for 12px text).

3x3/4x4로 변경하면 타일이 ~640-768px로 작아지고,
resize_long=960에 의해 실제로 1.3-1.5x 업스케일되어
12px 텍스트가 ~15px로 커진다. det 검출 한계에 가까운 텍스트를 더 잡을 수 있다.

- >= 1400px: 2x2 → 3x3
- >= 2400px: 3x3 → 4x4

### 7. det.rs — BOX_THRESH 완화 (0.6 → 0.5)

경계에 있는 텍스트 영역이 BOX_THRESH=0.6에서 탈락하는 경우가 있었다.
0.5로 낮춰 더 많은 후보를 rec 단계로 넘긴다.

### 8. rec.rs — BGR 채널 순서 수정

det.rs, cls.rs는 이전에 BGR로 수정되었지만, rec.rs는 RGB 순서 그대로였다.
PaddleOCR rec 모델은 BGR 입력으로 학습되었으므로 R↔B가 뒤바뀐 상태로 인식하고 있었다.

정규화가 채널 무관(`x/127.5 - 1`)이라도 채널 순서 자체가 다르면
conv 필터가 잘못된 채널 패턴을 보게 되어 인식률이 저하된다.

## 향후 개선 방향

현재 PaddleOCR PP-OCRv3 모델은 인쇄물/카메라 이미지로 학습되어 화면 텍스트(ClearType 서브픽셀 렌더링, 12-16px 시스템 폰트)에 최적화되지 않았다.
코드 수준 최적화(타일링, NMS, 임계값)는 한계에 가까우며, 인식률의 의미 있는 다음 단계는 모델 교체다.

후보:
- **PP-OCRv4 server 모델**: PP-OCRv3 대비 정확도 향상, 특히 작은 텍스트에 강함. ONNX 변환 후 동일 파이프라인에서 drop-in 교체 가능.
- **TrOCR (Transformer OCR)**: Microsoft의 Transformer 기반 OCR. 화면 텍스트/문서에 강하지만, 모델 크기가 크고 추론 속도가 느릴 수 있음.
- **Windows OCR API 병행**: 화면 텍스트에는 OS 내장 OCR이 더 정확할 수 있으며, ONNX 모델과 결과를 병합하는 하이브리드 접근도 가능.

det 전처리(그레이스케일 + 언샤프 마스킹)는 시도했으나 모델의 학습 분포(컬러 BGR + ImageNet 정규화)에서 벗어나 오히려 인식률이 하락했다.

## 수정 파일

- `app/src/ocr/mod.rs` — `detect()`, `recognize_boxes()` 분리, `det` 모듈 `pub(crate)` 공개
- `app/src/ocr/rec.rs` — BGR 채널 순서 수정, 테스트 추가
- `app/src/services.rs` — 타일 우선 det 병합→NMS→rec 파이프라인, 사후 휴리스틱 제거, 테스트
