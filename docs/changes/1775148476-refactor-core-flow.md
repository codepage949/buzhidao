## 배경

- `main.ts`에 OCR 결과 병합, 텔레그램 API 호출, 환경 변수 접근이 한곳에 몰려 있다.
- `server/main.py`는 요청마다 같은 `output.png`를 덮어써서 동시 요청 시 결과가 섞일 수 있다.

## 목표

- OCR 결과 병합 로직을 순수 함수로 분리해 테스트 가능하게 만든다.
- 텔레그램 호출 중복을 줄여 메시지 전송 흐름을 명확하게 만든다.
- OCR 서버에서 요청별 임시 파일을 사용해 동시 요청 안전성을 확보한다.

## 비목표

- 키보드 훅 동작 방식 변경
- OCR 모델 설정 변경
- 텔레그램 기능 추가

## 작업 계획

1. 클라이언트 쪽 OCR 후처리와 텔레그램 전송을 별도 모듈로 분리한다.
2. 서버 쪽 파일 저장과 OCR 추론을 헬퍼 함수로 분리한다.
3. 순수 함수 테스트와 서버 문법 검증으로 회귀를 확인한다.
