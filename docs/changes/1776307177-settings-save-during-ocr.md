# OCR 진행 중 설정 저장 UX 정리

## 배경

현재 설정 창은 OCR 진행 상태와 무관하게 열리고 저장할 수 있다.
실제 데이터 정합성은 Rust 쪽 락과 순차 처리로 크게 깨지지 않지만,
OCR 진행 중 언어를 바꾸면 기존 OCR 완료까지 재웜업이 대기하므로
사용자는 "저장" 직후 로딩 창이 예상보다 오래 떠 있는 상황을 만날 수 있다.

문제의 핵심은 경쟁 조건보다 UX다.
- 사용자는 저장이 멈춘 것처럼 느낄 수 있다.
- 언어 변경만 오래 걸리는데, 현재 UI에서는 이유가 드러나지 않는다.
- 반대로 저장 전체를 막으면 API 키, 모델명, 단축키 같은 즉시 저장 가능한 항목까지 같이 막히게 된다.

## 현재 구조

- `app/src/lib.rs`
  - OCR 실행 중 상태는 `CaptureRetryState.ocr_in_flight`와 단축키 `busy`로 관리한다.
  - 설정 저장 시 `source`가 바뀐 경우에만 `set_lang() + warmup()`이 실행된다.
- `app/ui/src/settings.tsx`
  - 현재는 OCR 상태를 읽지 않는다.
  - 저장 버튼 비활성화 조건은 `saving`뿐이다.

즉, 문제를 푸는 기준은 "OCR 진행 상태를 settings 창이 얼마나 알아야 하는가"로 정리된다.

## 선택지

### 1. 안내 문구만 추가

- 방법
  - 현재 구조를 유지하고, 언어 변경 필드 또는 저장 상태 문구에
    "OCR 진행 중 저장하면 현재 작업이 끝난 뒤 언어 변경이 적용될 수 있음"을 명시한다.
- 장점
  - 구현이 가장 단순하다.
  - Rust/프론트 상태 동기화를 새로 만들지 않아도 된다.
- 단점
  - 실제로 가장 불편한 순간에 저장 버튼을 계속 누를 수 있다.
  - 사용자가 현재 정말 OCR 중인지 알 수 없다.

### 2. OCR 진행 중에는 저장 버튼 전체 비활성화

- 방법
  - settings 창이 `ocr_in_flight` 또는 앱의 "OCR 작업 중" 상태를 구독하고,
    진행 중이면 저장 버튼을 비활성화한다.
- 장점
  - UX가 가장 단순하고 예측 가능하다.
  - "왜 오래 걸리나" 대신 "지금은 저장 불가"로 명확하게 제어한다.
- 단점
  - 언어 변경과 무관한 설정도 전부 같이 막힌다.
  - 실제 문제보다 제약이 크다.

### 3. OCR 진행 중에는 언어 변경 저장만 제한

- 방법
  - settings 창이 OCR 진행 상태를 알고,
    `source`가 기존 값과 달라졌을 때만 저장 버튼을 비활성화하거나 별도 경고를 띄운다.
  - 다른 필드만 바꾼 저장은 그대로 허용한다.
- 장점
  - 실제 문제 지점만 정확히 겨냥한다.
  - 불필요한 제약이 적다.
- 단점
  - "원본 설정값"과 "현재 편집값" 비교가 필요하다.
  - 상태/메시지 분기가 조금 늘어난다.

### 4. 저장은 허용하되, OCR 진행 중이면 대기 안내를 강하게 표시

- 방법
  - 저장 버튼은 누를 수 있게 두고,
    OCR 진행 중 + 언어 변경 저장일 때
    "현재 OCR이 끝난 뒤 언어 변경을 적용합니다" 상태 문구를 명시한다.
  - 필요하면 저장 버튼 라벨을 `저장 후 대기 중...` 같은 식으로 바꾼다.
- 장점
  - 기능 제약이 없다.
  - 현재 동작 모델과 가장 잘 맞는다.
- 단점
  - 대기 시간이 길면 여전히 답답하다.
  - 사용자가 "왜 지금 막지 않지?"라고 느낄 수 있다.

## 결정

이번 작업은 `2. OCR 진행 중에는 저장 버튼 전체 비활성화`로 간다.

- settings 창이 OCR 진행 상태를 조회/구독한다.
- OCR 진행 중이면 저장 버튼을 전체 비활성화한다.
- 안내 문구는 `OCR 진행 중에는 설정을 저장할 수 없습니다.`로 고정한다.
- OCR이 종료되면 비활성화가 자동으로 풀리게 한다.

이 방향을 택한 이유는 다음과 같다.
- 구현과 상태 모델이 가장 단순하다.
- 사용자가 저장 버튼을 반복 클릭하는 상황을 막을 수 있다.
- 언어 변경만 따로 구분하는 추가 분기 없이도 현재 UX 문제를 충분히 줄일 수 있다.

## 구현 스케치

1. Rust에 settings 창이 읽을 수 있는 OCR 진행 상태 커맨드/이벤트를 추가한다.
2. `settings.tsx`에서 초기 상태 조회 + 진행 상태 갱신 구독을 붙인다.
3. `ocrBusy`가 `true`이면 저장 버튼을 비활성화한다.
4. 하단 상태 문구에 `OCR 진행 중에는 설정을 저장할 수 없습니다.`를 표시한다.

## 테스트 방향

- 자동 테스트
  - 순수 함수로 `저장 가능 여부/상태 문구` 계산을 분리하면 UI 로직 테스트 가능
- 육안 검증
  - OCR 진행 중 settings 창 열기
  - 저장 버튼 전체 비활성화 확인
  - OCR 종료 후 버튼 자동 활성화 확인
